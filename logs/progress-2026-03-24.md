### Technical Analysis

**Root Cause of Hang:** Console handoff conflict
- Early console (`earlycon=uart8250,mmio32,0x05000000`) works
- Regular console (`console=ttyS0,115200`) initializes  
- `keep_bootcon` prevents early console from being disabled
- **Conflict:** Both consoles try to control the same UART simultaneously
- **Symptom:** UTF-8 replacement character (0xef 0xbf 0xbd = "�") indicates corruption

**The mmio vs mmio32 Issue:**
- `mmio` assumes 64-bit memory access (works on ARM64)
- `mmio32` forces 32-bit memory access (required for ARM32)  
- H6 UART registers need 32-bit access when kernel runs in AArch32 mode
- This was the critical missing piece that caused the initial single "[" output

### Next Fix Deployed
